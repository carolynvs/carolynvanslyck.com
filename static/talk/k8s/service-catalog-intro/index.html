Service Catalog in Five Minutes

- Who has used Service Catalog before? - Who has used a broker with Cloud Foundry before?

Open Service Broker API

* Standardizes how platforms (like K8s) can work with a cloud's services. * Came from Cloud Foundry. * Reuse the same broker! * Clouds with Brokers: Microsoft, Google, Amazon, IBM and more.
  • Data & Analytics – databases, ElasticSearch
  • Integration – Box, Twitter, SendGrid
  • Utilities – conversions, speech to text
  • Infrastructure – networks, volumes, routing
  • DevOps – monitoring, metrics, auto-scaling

Why is this a problem again?

Quick thought experiment of what it is like to provision entirely from k8s, and manage from k8s. vs provisioning pieces from the cloud portal or a cli.

Why Not Operators?

This is a not a unique problem and we aren't the only solution. Service Catalog takes advantage of all the existing brokers that were made for Cloud Foundry and lets us use them in K8s.
